# 네이버 블로그 초안 작성 스킬 (Tooly)

Tooly(`tooly.deluxo.co.kr`)로 외부 검색 유입을 만드는 네이버 블로그 글 **초안**을 생성한다.
실제 발행(이미지 캡처·경험담 보강·네이버 업로드)은 사람이 한다 — 이 스킬은 **발행 직전까지의 초안**만 만든다.

## 단일 출처 (반드시 먼저 읽기)

**`marketing/naver-blog-playbook.md`** 가 모든 규칙의 원본이다. 작성 전 반드시 읽고 그대로 따른다. 이 파일은 규칙을 요약만 하므로, 충돌 시 플레이북이 우선.

기존 모범 예시(톤·구조 참고): `marketing/naver-post-capital-gains-1household-12eok.md`(단일 케이스+경험담형), `marketing/naver-post-capital-gains-by-price.md`(비교형).

## 입력 (없으면 사용자에게 질문)

1. **타깃 키워드/검색 시나리오** — 가능하면 실데이터(GSC·네이버 유입검색어)에서 뜬 롱테일. 광범위 헤드 키워드는 지양.
2. **연결할 Tooly 페이지** — `/finance/...` 등. 그 페이지의 계산 로직·콘텐츠를 읽어 수치·근거를 정확히 인용.
3. **글 성격** — 단일 케이스(경험담형) / 비교형 / 가이드형. 같은 페이지로 이미 발행된 글이 있으면 **유사문서 회피**를 위해 다른 성격·다른 시나리오로.

## 산출물

`marketing/naver-post-<slug>.md` 파일 1개. 아래를 **모두** 포함한다(플레이북 섹션 3·4·9 기준):

- 제목: `[세부 키워드]+[숫자/조건]+[의도어]`, 30자 내외. 정식명+구어체 병기 고려.
- 본문 구조: 들어가며 → 🔑 결론부터(TL;DR) → 개념/공식(+출처) → 실제 계산 예시 → 경험담(1인칭 placeholder, 사람이 채울 자리 표시) → 케이스 비교 → FAQ 3~5 → 마무리.
- **계산 수치는 실제 페이지 로직으로 검증**한 값만 쓴다(추측 금지). 필요하면 node로 계산 확인.
- **스크린샷 마커**: `📸 [스샷 N] (화면 설명) — 여기 삽입`. 위치 = 설명 텍스트 직후. 우선순위 결과→입력→breakdown→비교→모바일. (플레이북 섹션 3)
- **Tooly 링크 2개**: 후반부(경험담·마무리)에 맥락 텍스트 링크 + UTM(`?utm_source=naver_blog&utm_medium=referral&utm_campaign=<slug>`). 딥링크 프리셋 활용 가능. (저품질 회피 — 섹션 9)
- **출처**: 국세청·홈택스·법령 등 + 기준일/개정 고지(세금 글 필수).
- **발행 메모 블록**(작성자용, 발행 시 삭제): 타깃 키워드, 스샷 체크리스트, 링크 위치, **해시태그 15~30개(주제 키워드 위주, `#내돈내산`·`#직접계산` 같은 범용/유행 태그 금지 — 섹션 4·9)**.

## 저품질·정확성 게이트 (산출 전 자가 점검)

- [ ] 순수 AI 톤 아님 — 경험담 자리(1인칭, 구체 수치·감정·헷갈린 점) 확보. "~하겠습니다" 남발 금지.
- [ ] 같은 블로그 기존 글과 유사문서 아님(다른 시나리오·구조).
- [ ] 키워드 스터핑 아님(핵심 키워드 본문 5~7회 이내).
- [ ] 세금/금융 수치 정확(페이지 로직 검증) + 기준일·개정 고지.
- [ ] 외부 링크 1~2개, 후반부, 맥락 텍스트 링크.

## 하지 않는 것

- 발행 자동화(네이버는 공식 발행 API 없음 + 자동/대량 포스팅 = 저품질 트리거).
- 사실과 다른 경험담 날조(자리만 표시, 실제 경험은 사람이).
- 프로그래매틱 양산(thin/doorway = noindex 대상).
