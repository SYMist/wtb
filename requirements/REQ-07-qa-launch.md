# REQ-07: QA, 최적화, 애드센스 삽입 및 런칭

**대응 주차**: Week 7
**담당**: 전원
**PRD 참조**: 섹션 5.5, 7, 9, 10

---

## 1. 목적

Phase 1 전체 기능의 품질을 검증하고, 성능 최적화, 애드센스 광고 코드 삽입, SEO 최종 점검을 거쳐 런칭한다.

---

## 2. 요구사항

### 2.1 QA / 크로스 브라우저 테스트

| ID | 요구사항 | 우선순위 | 인수 조건 |
|---|---|---|---|
| R-07-01 | 전체 10종 계산기의 계산 정확도 검증 | 필수 | 각 계산기별 최소 5개 테스트 케이스 통과 |
| R-07-02 | 데스크톱 브라우저 테스트: Chrome, Safari, Edge | 필수 | 레이아웃 및 기능 정상 동작 |
| R-07-03 | 모바일 브라우저 테스트: iOS Safari, Android Chrome | 필수 | 반응형 레이아웃, 터치 인터랙션 정상 |
| R-07-04 | 계산기 간 연결 흐름 테스트: CTA 클릭 → 파라미터 전달 → 값 복원 | 필수 | 킬러 3종 간 삼각 순환 + 일반 계산기 연결 |
| R-07-05 | 결과 공유 테스트: 링크 복사 → 새 브라우저에서 접속 → 값 복원 | 필수 | 동일한 계산 결과 재현 |
| R-07-06 | 검색 기능 테스트: 자동완성 동작, 결과 클릭 시 이동 | 필수 | |
| R-07-07 | 엣지 케이스: 0 입력, 음수, 극단값(30억 등), 빈 입력 | 필수 | 에러 없이 적절한 결과 또는 안내 표시 |

### 2.2 성능 최적화

| ID | 요구사항 | 우선순위 | 인수 조건 |
|---|---|---|---|

| R-07-11 | 이미지 최적화 (Next.js Image 컴포넌트 사용, 있는 경우) | 권장 | |

### 2.3 애드센스 광고 삽입

| ID | 요구사항 | 우선순위 | 인수 조건 |
|---|---|---|---|
| R-07-13 | 애드센스 승인 상태 확인 | 필수 | 승인 완료 상태 |
| R-07-14 | 상단 배너 광고 삽입: 모든 계산기 페이지 + 홈페이지 | 필수 | 728x90 또는 반응형 |
| R-07-15 | 인라인 광고 삽입: 계산 결과와 가이드 텍스트 사이 | 필수 | 반응형 |
| R-07-16 | 사이드바 광고 삽입: 데스크톱 계산기 페이지 | 필수 | 300x250, 모바일 비노출 |
| R-07-17 | 금지 영역 준수 확인: 입력 폼 ↔ 결과 사이에 광고 없음 | 필수 | |
| R-07-18 | 광고 로드 시 레이아웃 시프트 없음 (CLS) | 필수 | 광고 슬롯 최소 높이 예약 |
| R-07-19 | ads.txt 파일 배포 | 필수 | Vercel 루트에 ads.txt 존재 |

### 2.4 SEO 최종 점검

| ID | 요구사항 | 우선순위 | 인수 조건 |
|---|---|---|---|
| R-07-20 | 전체 페이지 title, meta description 고유성 확인 | 필수 | 중복 없음 |

| R-07-23 | sitemap.xml 전체 페이지 포함 확인 | 필수 | 10개 계산기 + 홈 + 카테고리 페이지 |
| R-07-24 | robots.txt 정상 동작 확인 | 필수 | 크롤링 허용, 사이트맵 경로 포함 |
