# REQ-03: 킬러 계산기 #1 - 연봉 실수령액 분석기

**대응 주차**: Week 3
**담당**: FE, UX/UI
**PRD 참조**: 섹션 3.1.2
**URL**: `/finance/salary-calculator`

---

## 1. 목적

연봉/월급 입력 시 4대보험 및 세금 공제 내역을 시각화하고, 연봉 구간별 비교를 제공하는 킬러 계산기를 구현한다. 첫 번째 킬러 계산기로서 차트 연동, 실시간 갱신, 계산기 간 연결의 기본 패턴을 확립한다.

---

## 2. 요구사항

### 2.1 입력

| ID | 요구사항 | 우선순위 | 인수 조건 |
|---|---|---|---|
| R-03-01 | 연봉/월급 전환 토글 | 필수 | 토글 시 입력값 자동 환산 (연봉÷12, 월급×12) |
| R-03-02 | 연봉 슬라이더 (범위: 2,000만~2억, 기본값: 4,000만) + 직접 입력 | 필수 | 슬라이더와 입력 필드 양방향 동기화 |
| R-03-03 | 비과세액 입력 (기본값: 0원) | 필수 | 비과세액만큼 과세 대상에서 제외 |
| R-03-04 | 부양가족 수 선택 (본인 포함, 기본값: 1명) | 필수 | 소득세 간이세액 계산에 반영 |
| R-03-05 | 20세 이하 자녀 수 선택 (기본값: 0명) | 필수 | 소득세 간이세액 계산에 반영 |
| R-03-06 | URL 쿼리 파라미터로 초기값 설정 (`?annual=50000000&dependents=2` 등) | 필수 | 외부에서 링크 접속 시 해당 값으로 초기화 |

### 2.3 출력

| ID | 요구사항 | 우선순위 | 인수 조건 |
|---|---|---|---|
| R-03-15 | 핵심 요약 카드: 월 실수령액 크게 표시 | 필수 | 결과 영역 최상단, 강조 표시 |
| R-03-16 | 공제 항목별 상세 테이블 (항목명, 금액, 비율) | 필수 | 6개 공제 항목 + 합계 |
| R-03-17 | 연간 환산 표시 (연 실수령액, 연간 총 공제액) | 필수 | 월 실수령액 × 12 |

## 4. 기술 참고

- 이 계산기에서 확립할 패턴: 슬라이더-입력 양방향 바인딩, 실시간 차트 갱신, 쿼리 파라미터 기반 상태 복원, CTA를 통한 계산기 간 데이터 전달
