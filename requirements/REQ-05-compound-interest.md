# REQ-05: 킬러 계산기 #3 - 투자 복리 계산기

**대응 주차**: Week 5
**담당**: FE, UX/UI
**PRD 참조**: 섹션 3.1.3
**URL**: `/finance/compound-interest`

---

## 1. 목적

정방향/역방향 두 가지 모드의 복리 계산과 적립식/거치식 비교, 인플레이션 반영을 통해 투자 의사결정을 지원하는 킬러 계산기를 구현한다.

---

## 2. 요구사항

### 2.1 입력

| ID | 요구사항 | 우선순위 | 인수 조건 |
|---|---|---|---|
| R-05-01 | 모드 전환 탭: 정방향("얼마를 모을 수 있나") / 역방향("목표까지 얼마가 필요한가") | 필수 | 탭 전환 시 입력 폼 변경 |
| R-05-02 | **정방향 입력**: 초기 투자금, 월 적립액, 예상 연 수익률(%), 투자 기간(년) | 필수 | 각 항목 슬라이더 + 직접 입력 |
| R-05-03 | **역방향 입력**: 목표 금액, 투자 기간(년), 예상 연 수익률(%) 또는 월 적립액 중 하나 | 필수 | 누락된 값을 역산 |
| R-05-04 | 인플레이션율 입력 (선택, 기본값: 0%, 범위: 0~10%) | 권장 | 체크박스로 활성화, 활성 시 슬라이더 표시 |
| R-05-05 | 적립식/거치식 비교 토글 (기본: 적립식만 표시) | 필수 | 활성 시 비교 결과 동시 표시 |
| R-05-06 | URL 쿼리 파라미터 초기값 (`?monthly={실수령액}` 등) | 필수 | 연봉 분석기에서 넘어온 값 반영 |

### 2.2 계산 로직

| ID | 요구사항 | 우선순위 | 인수 조건 |
|---|---|---|---|
| R-05-07 | 적립식 복리 계산: FV = PV×(1+r)^n + PMT×((1+r)^n - 1)/r | 필수 | 월 복리 기준 (연이율/12) |
| R-05-08 | 거치식 복리 계산: FV = PV×(1+r)^n | 필수 | 초기 투자금만 운용 |
| R-05-09 | 역방향 계산: 목표 금액에서 필요한 월 적립액 또는 수익률 역산 | 필수 | 수학적으로 정확한 역산 |
| R-05-10 | 인플레이션 적용 실질 수익 계산 | 권장 | 명목 금액에서 인플레이션 차감 |
| R-05-11 | 년도별 누적 데이터 생성 (원금, 수익, 총액, 실질 총액) | 필수 | 차트용 데이터 배열 |
| R-05-12 | 계산 로직 순수 함수 분리 (`lib/calculators/compound.ts`) | 필수 | |

### 2.3 출력

| ID | 요구사항 | 우선순위 | 인수 조건 |
|---|---|---|---|
| R-05-13 | **정방향 결과**: 최종 금액, 총 투자 원금, 총 수익, 수익률(%) | 필수 | 핵심 요약 카드 |
| R-05-14 | **역방향 결과**: 필요 월 적립액 또는 필요 수익률 | 필수 | 강조 표시 |
| R-05-15 | 적립식 vs 거치식 비교 테이블 (비교 모드 활성 시) | 필수 | 최종 금액, 수익, 수익률 비교 |
| R-05-16 | 인플레이션 적용 시: 명목 금액과 실질 금액 병기 | 권장 | 실질 금액에 별도 색상 표시 |
