# REQ-01: 프로젝트 초기 셋업 및 인프라 구성

**대응 주차**: Week 1
**담당**: 전원
**PRD 참조**: 섹션 1.1, 6, 9

---

## 1. 목적

개발 환경 구성, 도메인 확보, 배포 파이프라인 연결, 애드센스 승인 절차를 병렬로 시작한다. 이후 모든 개발의 기반이 되는 프로젝트 스캐폴딩을 완성한다.

---

## 2. 요구사항

### 2.1 도메인 및 배포

| ID | 요구사항 | 우선순위 | 인수 조건 |
|---|---|---|---|
| R-01-01 | 도메인 구매 (1순위 tooly.kr → 2순위 tooly.co.kr → 3순위 gettooly.com) | 필수 | 도메인 소유 확인 가능 |
| R-01-02 | Vercel 프로젝트 생성 및 GitHub 리포지토리 연결 | 필수 | main 브랜치 push 시 자동 배포 동작 |
| R-01-03 | 확보한 도메인을 Vercel 커스텀 도메인으로 연결 | 필수 | 도메인 접속 시 사이트 표시 |
| R-01-04 | HTTPS 자동 설정 확인 | 필수 | SSL 인증서 정상 적용 |
| R-01-05 | 임시 랜딩 페이지 배포 ("Tooly - 생활 계산기 & 도구 모음, 곧 출시됩니다") | 필수 | 도메인 접속 시 랜딩 페이지 표시 |

### 2.2 애드센스

| ID | 요구사항 | 우선순위 | 인수 조건 |
|---|---|---|---|
| R-01-06 | 임시 랜딩 페이지에 애드센스 사이트 인증 코드 삽입 | 필수 | HTML head에 애드센스 스크립트 존재 |
| R-01-07 | 애드센스에 새 사이트 등록 및 승인 요청 | 필수 | 애드센스 대시보드에서 사이트 심사 중 상태 확인 |

### 2.3 프로젝트 스캐폴딩

| ID | 요구사항 | 우선순위 | 인수 조건 |
|---|---|---|---|
| R-01-08 | Next.js 14 (App Router) + TypeScript 프로젝트 생성 | 필수 | `npm run dev`로 로컬 실행 가능 |
| R-01-09 | Tailwind CSS 설정 | 필수 | Tailwind 클래스 적용 시 스타일 반영 |
| R-01-10 | ESLint + Prettier 설정 | 필수 | `npm run lint` 정상 실행, 포맷터 적용 |
| R-01-11 | 프로젝트 디렉토리 구조 생성 | 필수 | 아래 구조대로 폴더 존재 |
