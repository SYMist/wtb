# 2차 미팅: 생활 계산기 차별화 전략

**일시**: 2026-04-09
**목적**: 포화된 생활 계산기 시장에서의 차별화 방안 도출
**참석**: PO, PM, UX/UI 디자이너, 프로덕트 디자이너, FE 엔지니어, BE 엔지니어, 마케터

---

## 논의

**PM**: 오늘 안건은 명확합니다. 생활 계산기는 이미 네이버, 은행 공식 사이트, 개인 블로그, 기존 도구 사이트 등 셀 수 없이 많은 대체재가 존재합니다. 우리가 이 시장에 들어가서 트래픽을 확보할 수 있는 차별점이 뭔지 논의합시다.

**마케터**: 먼저 현실 인정부터 합시다. "대출이자 계산기"를 검색하면 네이버 자체 계산기가 최상단에 뜨고, 그 밑에 은행 공식 사이트, 그 밑에 10년 넘은 개인 사이트들이 이미 자리 잡고 있어요. 단일 키워드로 정면 돌파하는 건 사실상 불가능합니다.

**PO**: 그러면 아이템 자체를 재고해야 하는 건 아닌가요? 이미 포화된 시장에 굳이 들어갈 필요가 있는지부터 의문입니다.

**마케터**: 아닙니다. 포화된 건 "단순 계산기" 시장이에요. 제가 실제로 상위 노출 사이트 20개 정도를 분석해봤는데 공통적인 약점이 있습니다. 첫째, 대부분 단일 기능 페이지예요. 대출이자 계산기 하나, 끝. 맥락이 전혀 없어요. 둘째, 결과만 보여주고 해석이 없습니다. 숫자만 나오고 "그래서 어떻게 하라는 건데?"에 대한 답이 없어요. 셋째, 모바일 UX가 열악합니다. 오래된 사이트일수록 반응형 대응이 안 되어 있고요. 넷째, 데이터 연동이 없어요. 계산기와 실제 시장 데이터가 완전히 분리되어 있습니다.

**프로덕트 디자이너**: 핵심을 짚으셨어요. 기존 계산기들의 문제는 "계산"에서 끝난다는 겁니다. 사용자가 대출이자를 계산하는 이유는 숫자를 보려는 게 아니라 의사결정을 하려는 거예요. "이 조건이면 감당할 수 있나?", "다른 옵션은 뭐가 있지?" 이런 질문에 답하는 사이트가 없습니다.

**UX/UI 디자이너**: 맞습니다. 저도 조사해봤는데 기존 계산기 사이트들의 사용자 여정은 이래요. 검색해서 계산기 페이지에 도착하고, 숫자 입력하고, 결과 확인하고, 바로 이탈합니다. 체류시간이 30초에서 1분이에요. 광고 수익 측면에서도 비효율적이고요. 우리가 노려야 하는 건 검색으로 유입된 다음에 계산하고, 결과와 함께 시각화와 비교를 보여주고, 관련 다른 계산기로 자연스럽게 이동시켜서 체류시간을 늘리는 흐름입니다.

**PO**: 구체적으로 어떤 형태를 말하는 건가요?

**프로덕트 디자이너**: 예를 들어 대출이자 계산기라면요. 기존 사이트는 금액, 이자율, 기간 입력하면 월 상환액 하나 출력하고 끝이에요. 우리는 같은 입력을 받되, 월 상환액과 함께 원리금균등 대 원금균등 대 만기일시 비교 차트를 보여주고, 총 이자 비교를 해주고, 현재 시중 금리 데이터를 연동해서 보여주고, "연봉 대비 상환 부담률"을 자동으로 계산해주는 거예요. 그리고 거기서 연봉 실수령액 계산기로 자연스럽게 연결되는 겁니다. 하나의 계산기가 아니라 계산기 간 흐름을 설계하는 거죠.

**FE 엔지니어**: 기술적으로 충분히 가능합니다. 계산기 간 파라미터를 URL query string이나 로컬 스토리지로 넘기면 돼요. 사용자가 대출 계산기에서 입력한 금액이 연봉 계산기에서 자동으로 참조되는 식이죠. 구현 복잡도도 크지 않습니다.

**PO**: 방향성은 좋은데, 이게 정말 SEO에서 먹히나요? 결국 검색 유입이 안 되면 아무리 좋은 UX도 의미 없잖아요.

**마케터**: 여기서 전략이 두 갈래로 나뉩니다. 첫 번째는 롱테일 복합 키워드 공략이에요. "대출이자 계산기"는 경쟁이 치열하지만, "전세대출 이자 연봉 대비 계산"이나 "아파트 매매 vs 전세 비교 계산" 같은 복합 키워드는 경쟁이 거의 없습니다. 우리 계산기가 이런 복합 시나리오를 다루면 자연스럽게 이 키워드를 잡게 돼요. 두 번째는 콘텐츠와 도구를 결합한 페이지입니다. "2026년 주택담보대출 금리 비교" 같은 정보성 콘텐츠 페이지 안에 계산기를 임베드하는 거예요. 콘텐츠로 유입시키고 도구로 체류시키는 구조입니다. 이건 1차 미팅에서 논의한 크롤링 트랙과도 맞물려요.

**BE 엔지니어**: 두 번째 전략이라면 크롤링 파이프라인의 우선순위가 올라가겠네요. 금리 데이터, 부동산 시세 데이터를 주기적으로 가져와서 콘텐츠 페이지를 자동 생성하고, 거기에 관련 계산기를 붙이는 구조가 되겠습니다.

**PO**: 두 전략이 상충하는 건 아니니 둘 다 가져갑시다. 하지만 Phase 1에서는 첫 번째 전략인 롱테일 복합 키워드에 집중하는 게 맞다고 봅니다. 콘텐츠 자동 생성은 파이프라인 구축이 필요하니까요.

**UX/UI 디자이너**: 차별화 요소를 하나 더 제안합니다. 기존 계산기 사이트들은 디자인이 구시대적이에요. 폼이 투박하고, 결과 표시가 텍스트 나열이고, 광고가 본문을 가리죠. 깔끔한 현대적 UI만으로도 상당한 차별화가 됩니다. 사용자 입장에서 같은 기능이면 예쁜 사이트를 다시 방문해요. 재방문율에 직접적인 영향을 줍니다.

**FE 엔지니어**: 거기에 인터랙티브 시각화를 더하면 더 좋습니다. 슬라이더로 금액을 조절하면 차트가 실시간으로 바뀌는 식이요. 기존 사이트들은 "계산" 버튼을 눌러야 결과가 나오는데, 우리는 입력과 동시에 결과가 갱신되게 하면 체감 차이가 확 ின니다.

**프로덕트 디자이너**: 정리하면 우리의 차별화 축은 세 가지입니다. 첫째, 시나리오 기반 흐름. 단일 계산이 아닌 의사결정 과정을 지원합니다. 둘째, 데이터 연동. 실시간 시장 데이터와 계산기를 결합합니다. 셋째, 현대적 UX. 인터랙티브 시각화, 실시간 갱신, 깔끔한 디자인입니다.

**PM**: 한 가지 우려가 있습니다. 차별화 요소를 많이 넣을수록 개발 공수가 늘어나요. 단순 계산기 30개를 빠르게 찍어내는 것과, 고퀄리티 계산기 5개를 만드는 것 중 초기 전략으로 뭐가 맞을까요?

**마케터**: SEO 관점에서는 페이지 수가 중요합니다. 5개 페이지로는 검색 노출 자체가 어려워요. 최소 15개에서 20개는 있어야 사이트 권위가 쌓이기 시작합니다.

**FE 엔지니어**: 절충안이 있습니다. 기본 계산기 템플릿을 잘 만들면 단순 계산기는 빠르게 찍어낼 수 있어요. 거기에 핵심 계산기 3개에서 5개만 시나리오 흐름과 시각화를 붙이는 겁니다. 이게 킬러 콘텐츠 역할을 하고, 나머지 단순 계산기들이 롱테일을 담당하는 구조예요.

**PO**: 좋습니다. 그 방향이 맞아요. 킬러 계산기로 사이트 정체성을 잡고, 나머지로 볼륨을 채우는 전략으로 갑시다.

**프로덕트 디자이너**: 킬러 계산기 후보를 제안하겠습니다. 첫째, 주택대출 종합 시뮬레이터. 매매가, 대출비율, 금리, 상환방식 비교를 한 화면에서 처리합니다. 둘째, 연봉 실수령액 분석기. 연봉 입력 시 4대보험, 소득세를 전부 시각화해줍니다. 셋째, 투자 복리 계산기. 적립식 거치식 비교, 인플레이션 반영, 목표 금액까지 역산을 지원합니다. 이 세 가지는 검색량도 높고 시나리오 확장도 풍부합니다.

**PM**: 반론 없으면 정리하겠습니다.

**(전원 동의)**

## 최종 결론

| 항목 | 결정 사항 |
|---|---|
| **차별화 전략** | 시나리오 기반 흐름 + 데이터 연동 + 현대적 UX |
| **킬러 계산기 (3종)** | 주택대출 시뮬레이터, 연봉 실수령액 분석기, 투자 복리 계산기 |
| **일반 계산기** | 템플릿 기반 15~20종 빠르게 확보 (롱테일 SEO) |
| **SEO 전략** | Phase 1: 롱테일 복합 키워드 / Phase 2: 콘텐츠-도구 결합 |
| **UX 원칙** | 실시간 갱신, 인터랙티브 차트, 계산기 간 데이터 연결 |
| **콘텐츠 원칙** | 계산 결과 + 해석/비교/맥락 제공 |
