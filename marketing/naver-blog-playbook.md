# 네이버 블로그 플레이북 (Tooly 백링크용)

Tooly(`tooly.deluxo.co.kr`)로 검색 유입과 백링크를 확보하기 위한 네이버 블로그 운영 가이드입니다. 포스팅 5~10개를 기준으로 작성되었습니다.

---

## 1. 목표와 전략

### 핵심 KPI
- **직접 유입**: 블로그 → Tooly CTA 클릭률 **10% 이상**
- **검색 노출**: 목표 키워드 각 포스팅당 최소 1개 상위 10위권 진입
- **도메인 권위**: 네이버 블로그 자체의 C-Rank 지수 누적 (장기)

### 네이버 검색 알고리즘 대응 포인트
네이버는 Google과 다르게 **C-Rank + DIA** 알고리즘을 씁니다.

| 요소 | 의미 | 실무 적용 |
|---|---|---|
| **C-Rank (Context)** | 출처·인용의 신뢰도 | 공식 출처(국세청, 한국은행, 통계청) 명시 |
| **C-Rank (Chain)** | 주제 일관성 | 한 블로그는 금융/생활정보 장르로 고정 |
| **C-Rank (Creation)** | 활동성 | 주 1~2회 꾸준한 발행 |
| **DIA (체류시간)** | 독자 만족도 | 목차·이미지·표로 스크롤 유도 |
| **DIA (정보성)** | 실질적 도움 | 계산 예시·수치·근거 포함 |
| **DIA (경험)** | 작성자의 실제 후기 | **1인칭 경험담 삽입** (→ 섹션 7 참고) |

### 2025~2026년 알고리즘 변화 포인트
- **AI 콘텐츠 탐지 강화**: 네이버가 AI/GPT 생성 콘텐츠를 판별하는 로직을 도입. 순수 AI 생성 글은 저품질 판정 위험이 있으므로 반드시 **개인 경험·수치·감정**을 덧붙여야 함
- **홈판(홈피드) 노출 부상**: 키워드 검색 노출 외에 네이버 홈 피드 노출이 방문자 수 핵심 채널로 부상. 저장·공유·댓글 수가 홈판 노출에 영향
- **체류시간·재방문율 가중치 증가**: 독자가 끝까지 읽고 저장하는 글이 알고리즘 점수를 더 받음
- **#내돈내산 #리얼후기 키워드 트렌드**: 사용자들이 선호하는 검색 패턴 변화 → 진성 경험담 콘텐츠가 노출 우선순위 상승

### 주의사항
- **외부 링크 남발 금지**: 한 포스팅당 Tooly 링크는 **1~2개**만 (자연스러운 맥락에서)
- **복붙 금지**: 각 포스팅 최소 70% 이상 original text
- **이미지 5~10장**: 네이버는 이미지 많은 글을 선호 (Tooly 결과 스크린샷 적극 활용)
- **제목에 숫자·년도**: "2026년", "5분 만에", "얼마인지"
- **AI 단독 작성 금지**: AI가 작성한 글을 그대로 올리면 저품질 판정 위험. 반드시 개인 경험·감정·구체적 상황을 추가해 인간 글쓰기 레이어를 씌울 것

## 2. 주제 선정 원칙

### 좋은 주제 = 아래 3가지 교집합
1. **검색 수요**: 네이버 검색량 있는 질문형 키워드
2. **Tooly 연결성**: Tooly 계산기로 답변 가능
3. **해설 가치**: 계산기만으론 부족한 맥락·예시·팁 제공 필요

아래 골격을 그대로 복사해서 쓰세요. 각 섹션에 최소 분량을 표시했습니다.

```markdown

- 독자의 고민/상황을 한 문장으로 공감
- 이 글을 읽으면 무엇을 알 수 있는지 1줄 요약
- [키워드]가 제목에서 본문 첫 3줄 안에 반복되도록

## 🔑 결론부터 (TL;DR) (100자)
- 핵심 답변 1~3줄
- 예: "연봉 5000만원이면 월 실수령액 약 **354만원**입니다. (2026년 기준, 부양가족 1인)"

## 1. [주제 개념 설명] (300~500자)
- 키워드가 무엇인지, 왜 중요한지
- 공식/기준 설명 (가능하면 표로 정리)
- **출처 명시**: 국세청, 한국은행, 통계청 등

## 2. 실제 계산 예시 (500~700자)
- 구체적 수치로 계산 과정 풀어쓰기
- **Tooly 결과 스크린샷** 1~2장 삽입
- 계산기 사용법 간단 안내
- 자연스러운 CTA:
  > 직접 해보고 싶다면 [Tooly 연봉 계산기](https://tooly.deluxo.co.kr/finance/salary-calculator)에서 숫자만 바꿔 확인할 수 있습니다.

## 3. 내 경험담 / 실제로 써봤더니 (200~400자) ★ 핵심

- Tooly 프로그래매틱 페이지 링크 가능 (연봉별 페이지)

## 6. 마무리 (100자)
- 핵심 요약 1줄
- 독자 행동 유도 (저장/공유/댓글)
```

- [ ] 클릭 유도하는 질문형/숫자형
- [ ] 낚시성 금지 (예: "충격", "대박" 등)

### 본문
- [ ] 1,500자 이상 (체류시간 확보)
- [ ] 이미지 5장 이상 (결과 스크린샷·표·그래프)

- [ ] **경험담 섹션 포함** (1인칭, 구체적 상황·수치·감정 서술)
- [ ] AI 단독 생성 티 나는 표현 제거 ("~하겠습니다", "~살펴보겠습니다" 남발 ✗)

### SEO
- [ ] 대표 키워드가 제목·첫 문단·소제목·마지막 문단에 분산 배치
- [ ] 연관 키워드 3~5개가 자연스럽게 본문에 등장

## 7. 경험담(체험기) 작성 원칙 ★ 신규

### 왜 경험담이 중요한가

네이버 DIA 알고리즘은 "사용자들의 선호" 또는 "작성자의 후기"처럼 **실제 경험이 담긴 문서**에 높은 점수를 부여합니다. 2025~2026년부터 AI 콘텐츠 탐지가 강화되면서, 개인 경험이 없는 정보 나열 글은 저품질로 판정될 위험이 커졌습니다.

핵심 원칙: **서툴러도 진솔한 경험 > 완벽하지만 AI 냄새 나는 글**

---

### 경험담 작성 3원칙

**① 구체적 수치와 상황을 넣어라**

나쁜 예:
> "연봉 계산기를 사용해보니 편리했습니다."

좋은 예:
> "연봉 협상이 끝나고 '그래서 실제로 얼마 받는 거지?' 싶어서 계산기를 돌려봤는데, 5,200만원에서 실수령액이 360만원대가 나왔을 때 솔직히 조금 실망했어요. 생각보다 많이 빠지더라고요. 그나마 식대 비과세 20만원을 빼고 나서 계산하니까 15만원 정도 더 나왔는데, 이 부분은 처음에 몰라서 헤맸습니다."

---

**② 실패·불편함·헷갈렸던 부분을 솔직하게 써라**

완벽한 후기는 광고처럼 보입니다. 알고리즘도, 독자도 신뢰하지 않습니다.

예시 표현:
- "처음엔 어디에 숫자를 넣어야 하는지 헷갈렸어요"
- "원리금균등이랑 원금균등이 뭔지 몰라서 두 번 검색했습니다"
- "생각보다 이자가 많이 나와서 당황했는데"
- "3개월 전에 알았더라면 다른 선택을 했을 것 같아서 아쉬웠어요"

---

**③ 감정·맥락·결과를 3단 구조로 써라**

```
[상황/배경] → [행동/경험] → [결과/느낌]
```

예시:
> 전세 계약 앞두고 '이 보증금이면 그냥 월세 사는 게 낫지 않나?' 진지하게 고민했어요. (상황)
> 전월세 전환 계산기에 보증금 3억, 전환율 4.5%로 돌려봤더니 월세 환산액이 112만원이 나왔고요. (행동)
> 주변 월세 시세가 85~90만원이라 전세가 유리하다는 결론이 나왔고 덕분에 고민이 정리됐습니다. (결과)

### 주제별 경험담 삽입 가이드

| 주제 | 경험담 소재 예시 |
|---|---|
| 연봉 실수령액 | 연봉 협상 후 통장 확인했을 때 느낌, 첫 월급명세서 보고 당황한 경험 |
| 주택대출 | 대출 상담 전 예상 납입액 계산해본 경험, 금리 1% 차이가 생각보다 크다는 걸 깨달은 순간 |
| 복리 계산 | 노후 준비 늦게 시작한 걸 후회하며 시뮬레이션 돌려본 경험 |
| 전세 vs 월세 | 이사 결정 전 직접 계산해서 결정한 과정 |
| 퇴직금 | 이직 시 퇴직금 예상액 몰라서 협상에서 손해 봤던 경험 |
| BMI | 검진 결과 보고 정상 범위인지 헷갈려서 찾아본 경험 |

### 경험담 분량과 위치

- **위치**: 정보 설명 섹션 직후, 계산기 CTA 직전에 삽입 (가장 자연스러운 흐름)
- **어투**: 구어체 허용 ("~했어요", "~더라고요", "~인 것 같아요") — 격식체보다 친근한 어투가 체류시간에 유리

## 8. AI 저품질 판정 회피 체크리스트

네이버는 2025년부터 AI 생성 콘텐츠 탐지 로직을 강화했습니다. 다음 항목을 확인하세요.

- [ ] 1인칭 경험 서술이 최소 1섹션 포함되어 있는가
- [ ] "~하겠습니다", "~살펴보겠습니다" 같은 AI 투의 격식체가 남발되지 않았는가
- [ ] 구체적 수치(금액, 날짜, 기간)가 경험담 안에 들어 있는가
- [ ] 실패·불편·헷갈림 같은 '부정적 뉘앙스'가 최소 1개 포함되어 있는가
- [ ] 본인의 의견·판단이 명확하게 드러나 있는가 ("저는 원금균등이 더 맞는 것 같더라고요")

## 부록. 첫 포스팅 샘플 오프닝 (1번 연봉 5000)

> **"연봉 5000 받는데 실수령액은 왜 이것밖에 안 되지?"** 한 번쯤 통장을 보며 생각해본 적 있을 거예요.
>
> 연봉 5000만원의 세전·세후 차이는 약 **천만원**에 달합니다. 소득세, 4대 보험, 부양가족 공제 여부에 따라 월 실수령액이 크게 달라지는데요. 오늘은 2026년 기준으로 **연봉 5000만원 실수령액**을 정확하게 계산하고, 세전·세후 구조를 쉽게 정리해드릴게요.
>
> 마지막에는 본인 연봉을 직접 넣어볼 수 있는 계산기도 소개합니다.
>
> ## 🔑 결론부터
> - 연봉 5000만원 → **월 실수령액 약 354만원** (부양가족 1인, 2026년 기준)
