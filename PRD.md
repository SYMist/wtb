## 1. 제품 개요

### 1.1 제품명

- **브랜드**: Tooly
- **서브 타이틀**: 생활 계산기 & 도구 모음
- **홈 페이지 title**: Tooly - 생활 계산기 & 도구 모음
- **도메인**: 1순위 tooly.kr → 2순위 tooly.co.kr → 3순위 gettooly.com

### 1.2 제품 정의

생활 속 다양한 계산과 변환을 제공하는 온라인 도구 모음 웹사이트. Google AdSense를 통해 수익화하며, 크롤링 기반 정보 콘텐츠를 결합하여 페이지 볼륨과 트래픽을 확보한다.

### 1.3 수익 모델

Google AdSense 광고 수익 (계정 보유 완료)

- 수익 공식: **페이지뷰 x CTR x CPC**
- 페이지뷰: SEO를 통한 검색 유입 + 계산기 간 흐름을 통한 내부 순환
- CTR/CPC: 금융 카테고리 중심의 고단가 키워드 공략

### 1.4 타겟 사용자

- 검색 엔진을 통해 특정 계산/변환이 필요한 한국어 사용자
- 연령대: 20~60대 (금융 계산기 사용자는 30~50대 중심)
- 기기: 모바일 60% 이상 예상

## 2. 핵심 전략

### 2.1 차별화 전략 (3축)

기존 계산기 사이트들의 공통 약점(단일 기능, 해석 부재, 열악한 UX, 데이터 미연동)을 공략한다.

| 축 | 설명 |
|---|---|
| **시나리오 기반 흐름** | 단일 계산이 아닌 의사결정 과정을 지원. 계산기 간 파라미터 전달을 통한 연속 여정 설계 |
| **실시간 데이터 연동** | 시중 금리, 환율 등 외부 데이터를 계산기에 반영. 정보 콘텐츠와 도구를 결합 |
| **현대적 UX** | 인터랙티브 시각화, 입력 즉시 결과 갱신(버튼 없음), 깔끔한 반응형 디자인 |

| Phase | 전략 | 설명 |

| Phase | 내용 | 상태 |

## 3. 기능 상세

### 3.1 킬러 계산기 (3종)

#### 3.1.1 주택대출 종합 시뮬레이터

**핵심 가치**: 여러 상환 방식을 한눈에 비교하여 대출 의사결정 지원

**입력 항목**:
- 매매가 (또는 대출금액 직접 입력)
- 자기자금 비율 (슬라이더)
- 연이자율 (직접 입력 + 시중 은행 금리 프리셋)
- 대출기간
- 거치기간
- 연봉 (DSR 계산용, 선택)

**출력 항목**:
- 핵심 요약: 월 상환액, 총 상환액, 총 이자
- 상환방식별 비교: 원리금균등 / 원금균등 / 만기일시 비교 테이블
- DSR (연봉 대비 상환 부담률)

**시각화**:
- 상환방식 비교 차트
- 월별 원리금 스택 바 차트 (원금/이자 색상 구분)

**데이터 연동**:
- 한국은행 기준금리 API
- 금융감독원 금융상품 비교 공시 데이터
- 갱신 주기: 일 1회 빌드 타임

**연결 흐름**:
- → 연봉 실수령액 분석기 (CTA: "정확한 실수령액이 궁금하다면?")
- ← 전월세 전환 계산기, 양도소득세 계산기, 평수 변환기

**하단 콘텐츠**: "2026년 주택담보대출 금리 현황" 등 정보성 텍스트

#### 3.1.2 연봉 실수령액 분석기

**핵심 가치**: 공제 항목 시각화와 연봉 구간별 비교를 통한 소득 이해 지원

**입력 항목**:
- 연봉 또는 월급 (슬라이더 + 직접 입력)
- 비과세액
- 부양가족 수
- 20세 이하 자녀 수

**출력 항목**:
- 월 실수령액
- 공제 항목별 금액 (국민연금, 건강보험, 장기요양보험, 고용보험, 소득세, 지방소득세)
- 연봉 구간별 실수령액 비교

**시각화**:
- 공제 항목 파이 차트
- 연봉 구간 비교 그래프

**데이터**:
- 4대보험 요율, 소득세 간이세액표 (정적 데이터, 연 1회 갱신)

**핵심 UX 포인트**:
- 슬라이더 조작 시 모든 수치와 차트 실시간 갱신 (계산 버튼 없음)

**연결 흐름**:
- → 투자 복리 계산기 (CTA: "실수령액에서 매월 투자 가능한 금액은?")
- → 주택대출 시뮬레이터 (CTA: "이 연봉으로 대출 가능한 금액이 궁금하다면?")
- ← 퇴직금 계산기, 부가세 계산기

#### 3.1.3 투자 복리 계산기

**핵심 가치**: 정방향/역방향 계산과 적립식/거치식 비교를 통한 투자 의사결정 지원

**입력 항목**:
- 모드 선택: 정방향("얼마를 모을 수 있나") / 역방향("목표까지 얼마가 필요한가")
- 초기 투자금
- 월 적립액
- 예상 수익률
- 투자 기간
- 인플레이션율 (선택)
- 목표 금액 (역방향 모드)

**출력 항목**:
- 최종 금액 (명목/실질)
- 총 투자 원금
- 총 수익
- 적립식 vs 거치식 비교
- 역산 결과 (필요 월 적립액 또는 수익률)

**시각화**:
- 원금 누적선 + 수익 포함 누적선 라인 차트
- 인플레이션 적용 실질 수익 선
- 적립식/거치식 비교 차트

**연결 흐름**:
- → 주택대출 시뮬레이터
- ← 연봉 실수령액 분석기, 예적금 이자 계산기, 환율 변환기

### 4.2 URL 규칙

- canonical 태그 및 메타 데이터 한글로 설정

### 4.3 계산기 간 흐름 (허브 앤 스포크)

킬러 계산기 3종을 허브로, 일반 계산기가 위성으로 연결되는 구조.

```
                    ┌─ 전월세 전환
                    ├─ 양도소득세
  주택대출 시뮬레이터 ◄─┤
        │           └─ 평수 변환
        │
        ▼
  연봉 실수령액 분석기 ◄─┬─ 퇴직금
        │              └─ 부가세
        │
        ▼
  투자 복리 계산기 ◄─────┬─ 예적금 이자
        │              └─ 환율 변환
        │
        └──────► 주택대출 시뮬레이터 (삼각 순환)
```

**연결 방식**:
1. **컨텍스트 CTA**: 결과 영역 내 맥락에 맞는 버튼. 입력값을 쿼리 파라미터로 전달
2. **관련 계산기 섹션**: 페이지 하단 "함께 많이 사용하는 계산기" 카드 3~4개

## 8. Phase 2: 데이터 포털 + 계산기 결합

### 8.1 지향 모델

tradingeconomics.com 스타일 **한국 특화 금융·생활 데이터 포털**. 계산기 단독 사이트가 아니라 "데이터 조회 → 현재 수치 확인 → 내 상황에 대입해 계산" 3단 여정으로 확장한다.

**tradingeconomics 분석 인사이트**:
- **데이터 축적이 곧 해자**: 1959년부터 60년치 시계열 데이터가 개인이 만든 사이트의 가치 핵심
- **2차원 URL 매트릭스**: `국가 × 지표` 조합으로 수백 페이지 자동 생성 (프로그래매틱 SEO 정점)
- **정보 밀도**: 단일 페이지에 Hero 수치 / 차트 / 테이블 / 서술 / 내부 링크 네트워크가 고밀도로 압축
- **계산기 없음**: 이 지점이 Tooly의 차별점 — 데이터 + 시뮬레이션을 결합 가능

### 8.2 데이터 페이지 표준 템플릿

모든 데이터 페이지는 아래 7블록 구조를 따른다.

```
┌────────────────────────────────────────┐
│ 1. Hero: 최신 수치 + 전월/전년 대비 변화  │
│    "기준금리 3.25% (2026-04, 전월 동일)"  │
├────────────────────────────────────────┤
│ 2. 시계열 차트 (1M / 6M / 1Y / 5Y / All)│
├────────────────────────────────────────┤
│ 3. 이력 테이블 (월별/연별, 최근 12~24행) │
├────────────────────────────────────────┤
│ 4. 관련 지표 비교 표 (예: 기준금리 vs 시중│
│    예금 금리 vs 대출 금리)               │
├────────────────────────────────────────┤
│ 5. 서술형 해설 (300~500자 + JSON-LD FAQ) │
│    - 지표 정의                           │
│    - 현재 수치 맥락                       │
│    - 과거 최고/최저 기록                  │
├────────────────────────────────────────┤
│ 6. 계산 시뮬레이션 CTA                    │
│    "이 금리로 3억 대출받으면? [계산]"     │
├────────────────────────────────────────┤
│ 7. 관련 데이터·계산기 링크 네트워크         │
└────────────────────────────────────────┘
```

### 8.3 데이터 도메인 & URL 구조

| 도메인 | URL 패턴 | 예시 페이지 | 데이터 소스 |
|---|---|---|---|
| **금리** | `/data/rates/[type]` | `/data/rates/base` (기준금리), `/data/rates/mortgage`, `/data/rates/deposit` | 한국은행 ECOS API |
| **환율** | `/data/exchange/[pair]` | `/data/exchange/usd-krw`, `/data/exchange/jpy-krw` | 한국수출입은행 API |
| **세금** | `/data/tax/[type]/[year]` | `/data/tax/salary-tax/2026`, `/data/tax/vat/2026` | 국세청 고시 |
| **통계** | `/data/stats/[indicator]` | `/data/stats/min-wage`, `/data/stats/cpi` | KOSIS |
| **부동산** | `/data/real-estate/[region]` | `/data/real-estate/seoul-gangnam` | 국토교통부 실거래가 |

각 축이 독립적으로 확장 가능. 예: `/data/rates/mortgage/[period]`처럼 기간 차원을 추가하면 페이지 폭발적 증가 (프로그래매틱 SEO).

### 8.4 계산기 ↔ 데이터 양방향 연결

**데이터 → 계산기** (데이터 페이지 하단 CTA):
- `/data/rates/mortgage` → "현재 금리 4.2%로 3억 대출 시 → 월 상환액 계산하기"
- `/data/exchange/usd-krw` → "현재 환율로 $1000 환전하기"
- `/data/tax/salary-tax/2026` → "2026년 세율로 내 실수령액 계산하기"

**계산기 → 데이터** (계산기 페이지 보조 정보):
- `/finance/loan-calculator` → "📊 현재 주택대출 금리 추이 보기" 링크
- `/finance/salary-calculator` → "📊 2026년 소득세율 변경사항 보기" 링크

### 8.5 데이터 축적 전략

**핵심 원칙**: 오늘부터 매일 fetch하면 1년 후 1년치 시계열 데이터가 생긴다. 축적이 곧 자산.

1. **일 1회 Cloudflare Cron** → `scripts/fetch-*.ts` 실행
2. **결과를 `data/time-series/*.json`에 append** (Git 저장 또는 KV)
3. **빌드 시 시계열 전체를 SSG 페이지에 주입**
4. **6개월 누적 시**: 차트 충분, 해설에 "지난 6개월 평균 X, 최고 Y" 같은 맥락 자동 생성

### 8.6 우선 구현 순서

| 순위 | 페이지 | 이유 |
|---|---|---|
| 1 | `/data/rates/base` (기준금리) | 단일 수치, 한국은행 ECOS 간단, 대출/예금 계산기와 직결 |
| 2 | `/data/exchange/usd-krw` | 이미 `exchange-rates.json` 자동화 TODO 존재 |
| 3 | `/data/rates/mortgage` | 주택대출 시뮬레이터의 금리 프리셋과 양방향 연결 |
| 4 | `/data/stats/min-wage` | 2026 최저임금 키워드 시즌성 트래픽 |
| 5 | `/data/tax/salary-tax/2026` | 연봉 실수령액 계산기 보조, 연 1회 갱신으로 관리 부담 적음 |

### 8.7 현실 체크

- tradingeconomics는 **10년 이상** 축적된 자산. 단시간 모방 불가
- **차별화 포인트**: 글로벌 일반 데이터 → Tooly는 한국 세법/4대보험/전월세 등 **현지화 깊이**가 해자
- **솔로 개발 가능 범위**: Phase 2a(금리/환율 2~3개)를 먼저 완성하고 트래픽 데이터로 확장 여부 결정
